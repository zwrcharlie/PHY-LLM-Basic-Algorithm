# System vs. User（实验室核心系统与研究人员）

## 零基础含义解释
System vs. User就像**实验室核心设备与研究人员个人实验**的区分。System进程是实验室维持基本运行的关键设备（如粒子加速器），User进程是研究人员运行的具体实验任务。

## 物理场景类比解释
在粒子物理实验中：
- System进程相当于**实验室核心设施**：
  - 管理探测器供电系统（如UPS电源）
  - 维持低温超导环境（如液氦冷却系统）
  - 监控辐射安全（如安全联锁系统）
- User进程相当于**具体实验任务**：
  - 数据采集程序
  - 数据分析脚本
  - 模拟计算任务

## 基本配置
```bash
# 查看系统进程（核心设备）
ps -eo user,pid,cmd --user root | head -n 10  # Linux查看系统进程

# Windows查看系统服务
sc query type= service  # 查看系统服务状态
```

## 物理系学生常见应用场景
- 监控超导磁体供电状态
- 区分实验数据采集与系统日志
- 管理低温控制系统与模拟计算任务

## 算法应用（物理实验场景）
```bash
# 区分系统与用户进程
ps -eo user,pid,cmd --sort user \
| grep -E "root|alice"  # 过滤系统与用户进程
```

## VSCode操作指引
1. 安装Process Explorer插件：
   - 提供系统/用户进程分类视图
   - 支持核心设备状态监控
   - 显示进程资源占用详情
2. 在"Processes"视图过滤系统/用户进程：
   - 点击"Filter"按钮
   - 选择"System/User"过滤模式
   - 选择特定用户（如alice）查看实验任务
3. 使用"Filter"功能按用户筛选实验任务：
   - 输入用户名过滤进程列表
   - 保存常用过滤配置
   - 组合过滤条件（用户+CPU占用）
4. 右键系统进程选择"Properties"查看核心设备状态：
   - 查看设备型号/固件版本
   - 监控设备温度/电压
   - 查看设备日志信息
5. 安装Python Environments管理器：
   - 可视化管理系统Python环境
   - 快速切换实验用Python版本
6. 使用"Performance Monitor"跟踪核心设备负载
